fs_read

Read files and list directories

Explore the filesystem with explicit modes (tree, list, content) and pagination. Automatically detects file vs directory in auto mode, and returns line-numbered content with checksums for files.

Input Schema

Parameter Type Description
path required string Relative path to file or directory. Use "." for root, "docs/" for directories.
mode optional enum Exploration mode for directories and files.
auto tree list content
Default: auto
limit optional number Max entries returned for directory listings.
Default: 100
offset optional number Skip the first N entries for directory listings.
Default: 0
lines optional string Limit file reading to specific lines. Format: "10" or "10-50".
depth optional number Directory listing depth (1-20).
Default: 1
details optional boolean Include file sizes and modification times.
Default: false
types optional string[] Filter by file extensions: ["ts", "js"]
glob optional string Glob pattern filter: "**/*.ts"
exclude optional string[] Patterns to exclude: ["**/test/**"]
respectIgnore optional boolean Respect .gitignore files.
Default: true

Output Schema

{
  "success": boolean,
  "path": string,
  "type": "directory" | "file",
  // For directories:
  "entries": [{ path, kind, size?, modified?, children? }],
  "summary": string,
  "stats": {
    "returned": number,
    "total": number,
    "offset": number,
    "limit": number,
    "hasMore": boolean,
    "truncated": boolean
  },
  // For files:
  "content": {
    "text": string,       // with line numbers
    "checksum": string,   // for safe editing
    "totalLines": number,
    "range?": { start, end },
    "truncated": boolean
  },
  "hint": string
}

fs_write

Create and update file content

Create new files or update existing ones with line-based targeting. Supports dry-run mode to preview changes before applying. Uses checksums to prevent overwriting concurrent modifications.

Input Schema

Parameter Type Description
path required string Relative path to file. Parent dirs created automatically.
operation required enum The operation type to perform.
create update
action required for update enum What to do with targeted content.
replace insert_before insert_after delete_lines
lines required for update string Target lines. Format: "10" or "10-15".
content conditional string Content to write. Required except for delete_lines.
checksum optional string Expected checksum from fs_read. Prevents stale overwrites.
dryRun optional boolean Preview changes without applying.
Default: false
createDirs optional boolean Create parent directories if missing.
Default: true
ensureTrailingNewline optional boolean Ensure file ends with newline (POSIX).
Default: true

Output Schema

{
  "success": boolean,
  "path": string,
  "operation": "create" | "update",
  "applied": boolean,       // false if dryRun
  "result": {
    "action": string,       // e.g. "replaced", "created"
    "linesAffected": number,
    "newChecksum": string,
    "diff?": string         // unified diff
  },
  "error?": { code, message, recoveryHint? },
  "hint": string
}

fs_search

Find files by name and search content

Powerful search combining filename matching (fuzzy) and content search. Supports literal text, regex patterns, and fuzzy matching. Includes preview mode and pagination for large result sets.

Input Schema

Parameter Type Description
path required string Starting directory. Use "." for all mounts.
query required string Search term for filenames and content.
target optional enum What to search.
all filename content
Default: all
preview optional boolean Lightweight output with per-file match counts only.
Default: false
patternMode optional enum How to interpret the query.
literal regex fuzzy
Default: literal
caseInsensitive optional boolean Ignore case in content search.
Default: false
wholeWord optional boolean Match whole words only.
Default: false
multiline optional boolean Allow matches spanning multiple lines.
Default: false
types optional string[] Filter by file type: ["ts", "md"]
glob optional string Glob pattern filter: "**/*.ts"
exclude optional string[] Patterns to exclude from search.
depth optional number Max directory traversal depth (1-20).
Default: 5
maxResults optional number Search cap for matches (1-1000).
Default: 100
limit optional number Max results returned per section.
Default: 50
offset optional number Skip the first N results per section.
Default: 0
context optional number Context lines around content matches (0-20).
Default: 3
respectIgnore optional boolean Respect .gitignore files.
Default: true

Output Schema

{
  "success": boolean,
  "query": string,
  "target": "all" | "filename" | "content",
  "results": {
    "byFilename": [{ path, score, matchIndices }],
    "byContent": [{
      "path": string,
      "matches?": [{
        line, endLine, matchCount, text,
        context: { before, match, after }
      }],
      "matchCount?": number // preview=true
    }]
  },
  "stats": { filenameMatches, contentMatches, filesSearched },
  "page": {
    "limit": number,
    "offset": number,
    "byFilename": { returned, total, hasMore },
    "byContent": { returned, total, hasMore }
  },
  "truncated": boolean,
  "hint": string
}

fs_manage

Structural filesystem operations

Perform structural operations on files and directories including delete, rename, move, copy, mkdir, and stat. Supports recursive operations and force overwrite options.

Input Schema

Parameter Type Description
operation required enum The structural operation to perform.
delete rename move copy mkdir stat
path required string Source file or directory path.
target conditional string Destination path. Required for rename/move/copy.
recursive optional boolean For delete: remove directory contents. For mkdir: create parent directories. For copy/move: include subdirectories.
Default: false
force optional boolean Overwrite if target exists.
Default: false

Output Schema

{
  "success": boolean,
  "operation": string,
  "path": string,
  "target?": string,
  "stat?": {               // for stat operation
    "size": number,
    "modified": string,    // ISO date
    "created": string,     // ISO date
    "isDirectory": boolean
  },
  "error?": { code, message },
  "hint": string
}
